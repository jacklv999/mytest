1. Traditional algorithm

Kmeans

Doing k-means step by step:

  • 选择初始化的 个样本作为初始聚类中心 ;
  • 针对每个样本 计算它到 个聚类中心的距离并将其分到距离最近的类；
  • 针对每个类别 ，重新计算它的聚类中心 ；
  • 重复操作，直到达到某个中止条件（迭代次数、最小误差变化等）;

优点

  • 处理大数据集的时候，该算法可以保证较好的伸缩性；
  • 当簇近似高斯分布的时候，效果非常不错；
  • 算法复杂度低。

缺点

  • K 值需要人为设定，不同 K 值得到的结果不一样；
  • 对初始的簇中心敏感，不同选取方式会得到不同结果；
  • 对异常值敏感；
  • 不适合太离散的分类、样本类别不平衡的分类、非凸形状的分类。

调优与改进

  • 数据预处理: 数据归一化，数据标准化, 异常值检测

  • 核函数:

    • 普通K-means 假设数据具有一样的先验概率并呈现球形分布。

    • 面对非凸的数据分布形状时我们可以引入核函数来优化。

      • 通过非线性映射将数据映射到高维空间并聚类

  • K-means++: 随机选一个初始点, 计算所有点到该点的距离, 并以的概率选择其余点为第二个聚类中心, 不断重复直到选出的中心点;

DBSCAN

Define some term before introduce this algorithm:

  • 邻域：对样本和距离，的邻域是指所有与距离不大于的样本集;
  • 核心对象：若的邻域中至少包含MinPts个样本，则为核心对象;
  • 密度直达：若在的邻域中，且是核心对象，则称由样本密度直达;
  • 密度可达：对和，若存在序列，其中，，并且由密度直达，则称与密度可达；

Doing DBSCAN step by step:

  • 遍历所有样本，找出所有满足邻域距离的核心对象的集合；
  • 任意选择一个核心对象，找出其所有密度可达的样本并生成聚类簇；
  • 从剩余的核心对象中移除2中找到的密度可达的样本；
  • 从更新后的核心对象集合重复执行2-3步直到核心对象都被遍历或移除。

优点：

  • 可对任意形状的稠密数据集进行聚类，(K-Means只适用于凸数据集);
  • 可以在聚类的同时发现异常点，对数据集中的异常点不敏感。
  • 聚类结果没有偏倚(初始值对K-Means结果有很大影响

缺点：

  • 如果样本集的密度不均匀、聚类间距差相差很大时，聚类质量较差;
  • 如果样本集较大时，聚类收敛时间较长;
  • 调参相对于传统的K-Means之类的聚类算法稍复杂;
Hierarchical clustering

假设有 n 个待聚类的样本，对于层次聚类算法，它的步骤是：

  • 步骤一：（初始化）将每个样本都视为一个聚类；
  • 步骤二：计算各个聚类之间的相似度；
  • 步骤三：寻找最近的两个聚类，将他们归为一类；
  • 步骤四：重复步骤二，步骤三；直到所有样本归为一类。

NOTE: 相似度的衡量

  • Average Linkage: 把两集合中的点两两的距离全部放在一起求均值;
  • Complete Linkage: 取两个集合中距离最远的两个点的距离;
  • Single Linkage: 取两个类中距离最近的两个样本的距离为两集合相似度;

优点：

  • 一次性得到聚类树，后期再分类无需重新计算；
  • 可以发现类别的层次关系。

缺点：

  • 计算复杂度高，不适合数据量大的；
  • 算法很可能形成链状。

2. New algorithm

Hierarchical Dirichlet Process

两层Bayes模型, 先随机分配, 根据已分配的数据推断该部分的数据分布,

  • 随机分配初始点
  • 根据已分配的点推断该数据簇的分布;
  • 新的点按归属于各个数据簇分布的概率分配至不同数据簇
  • 不断重复
MeanShift

Mean Shift（均值漂移）是基于密度的非参数聚类算法，其算法思想是假设不同簇类的数据集符合不同的概率密度分布，找到任一样本点密度增大的最快方向（最快方向的含义就是Mean Shift），样本密度高的区域对应于该分布的最大值，这些样本点最终会在局部密度最大值收敛，且收敛到相同局部最大值的点被认为是同一簇类的成员。

  • 随机指定区域中心及半径

  • 计算计算中心的偏移向量

  • 按偏移向量移动中心;

  • 不断重复, 直至不再移动即得到当前簇密度最大的点作为;

  • 若当前簇中心与其它簇中心的距离小于阈值则合并, 否则作为新的聚类中心;

  • 重复1、2、3、4、5直到所有的点都被标记访问;

Affinity Propagation

AP算法的基本思想是将全部样本看作网络的节点，然后通过网络中各条边的消息传递计算出各样本的聚类中心。

聚类过程中，共有两种消息在各节点间传递，分别是吸引度( responsibility)和归属度(availability) 。AP算法通过迭代过程不断更新每一个点的吸引度和归属度值，直到产生m个高质量的Exemplar（类似于质心），同时将其余的数据点分配到相应的聚类中。

在AP算法中有一些特殊名词：

  • Similarity：点和点的相似度记为，用欧氏距离(负值)来计算;
  • Preference：点的参考度称为或，取相似度值的中值。
  • Responsibility：用来描述点适合作为数据点的聚类中心的程度;
  • Availability：用来描述点选择点作为其聚类中心的适合程度;

算法过程:

  • 算法初始，这两个矩阵均初始化为0矩阵;

  • 更新吸引度矩阵: 比其它点近多少

  • 更新归属度矩阵;

  • 根据衰减系数λ对两个公式进行衰减;

  • 重复步骤2/3/4直至矩阵稳定或者达到最大迭代次数，算法结束.

优点

  • 无需指定聚类“数量”参数
  • 明确的质心，而不是由多个数据点求平均而得到的聚类中心
  • 初始值不敏感。多次执行AP聚类算法，得到的结果是完全一样的

缺点

  • 算法复杂度较高，为, (不适用于大数据, 1000以上);
  • 需要手动指定Preference和Damping factor;
  • 同样需要对数据做归一化!
