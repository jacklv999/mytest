2. 关系抽取

文本中包含大量知识, 关系抽取则是从文本中自动化抽取事实知识.

一个完整的关系抽取系统通常包括:

  • 命名实体识别
  • 实体连接(entity linking)
  • 关系分类

其中, 关系分类是最核心, 最有挑战的部分.

3. 经典的关系抽取

预定义关系类型, 然后使用大量标注样本训练模型进行关系抽取. 模型的发展趋势为:

  • Pattern mining: 利用人工模板;

  • Feature-based method: 利用ML识别word embedding, 然后构建分类模型;

  • Kernel-based methods: 利用SVM类的kernel方法构建模型;

  • Graphic models: 利用图理论, 消息传递和接受的方式;

  • Neural models

4. 问题与挑战

实际运用中存在很多挑战:

  • 大规模数据的获取十分昂贵;
  • 关系的长尾现象严重;
  • 许多事实只是隐藏在更复杂的上下文中;
  • 关系的不断增长;

5. 解决方案

大规模监督数据的挑战

  • 解决方案: 通过远程监督的方式, 自动获取数据

    即对给定小样本标注, 自动从网络中获取相同实体的文本, 然后将所有文本一并作为标注数据训练模型;

  • 缺陷: 引入噪音数据

  • 改进: 降噪方式

    • Multi-instance multi label: 将具有相同entity pair的句子视为bag, 从bag中学习;

      • 使用 “at least one” 假设, 其中至少有一个句子正确;

    • sentence-level attention: 赋予bag中句子不同的权重;

    • RL: 去除bag中不靠谱的句子

关系的长尾现象严重

解决方案: Few-shot learning

  • Metric learning: 学习语义度量

    • 在大规模标注数据集中学习语义度量方式, 在Few Shot learning训练集中度量语义关系, 在测试集中根据学到的度量方式

    • 示例:

      • 首先, 使用标注数据学习encoder, 用sentence embedding

      • 其次, 输入新的关系类型, , , 根据每个类别的五个训练样本得出embedding, 然后对向量平均, 用于定义该类的代表向量空间

      • 最后, 对测试集, encoding 测试集样本 , 然后判断其最接近的类别.

  • Meta learning: MAML: model agnostic meta learning

    • 学习对新任务的参数初始化

问题与改进:

  • 易受噪音样本影响

    • 解决方案: 加入注意力机制(instance-level & feature level)

    • 示例:

      • instance level: 利用 query 对样本做attention
      • feature level: 利用CNN对label做attention

  • 领域迁移: 测试集和训练集相差可能很大

    • 尚无解决方案

  • “以上都不是” 任务: 可能样本不属于任何一类

    • 尚无解决方案
文档关系抽取: 许多事实只是隐藏在更复杂的上下文中
  • 使用文档级别的监督训练集
新关系和新实体

Relation discovery:

  • 在无监督的语料中聚类出可能的关系

  • 引入 relation Siamese network

    • 判断两个句子是否表达同一种关系
    • 用作聚类算法中的距离度量

  • Bootstrapping system

    • 对于一个新的关系类型, 给定少量的标注数据

    • 以迭代的方式, 从海量的无监督数据中手机更多的相同关系样本

      • 通过远程监督, 收集candidate, 使用relation Siamese network过滤
      • 使用新样本fine-tune, 再使用新模型收集candidate, 然后重复

    • 然后再训练模型
