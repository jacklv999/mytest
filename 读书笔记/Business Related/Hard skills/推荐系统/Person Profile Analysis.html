User Profile Analysis

用户画像分为User Persona和User Profile两种, 前者为产品经理工作的重点, 后者是数据分析的重点, 主要由各种标签构成.

1. 用户画像作用

在互联网、电商领域用户画像常用来作为精准营销、推荐系统的基础性工作，其作用总体包括：

  • 精准营销：分析产品的潜在用户和用户的潜在需求，针对特定群体，精准营销;
  • 推荐系统：以用户画像为基础构建推荐系统，实现个性化推荐;
  • 风险检测: 检测用户的风险, 及时控制用户风险;
  • 广告系统: 用于广告推送和广告建模, 分析等;
  • 数据分析: 进行经营数据分析, 赋能企业经营

3. 精准营销

用户画像在精准营销中的主要作用在于找到和分析现有客户及潜在客户, 简单说, 我们需要做:

  • 现存客户:

    • 内容: 现存客户是谁/客户偏好和行为模式/哪类客户的价值最高
    • 目的: 维系客户/提高粘性/提高交易转化率

  • 潜在客户:

    • 内容: 潜在客户是谁/客户偏好/找到潜在客户的渠道/获客成本等
    • 目的: 如何挖掘潜在客户

5. Raw Data

按数据内容可以划分为:

  • 人口属性：地域、年龄、性别、教育程度、职业、收入、生活习惯、消费习惯等；
  • 产品行为：产品类别、活跃频率、产品喜好、产品驱动、使用习惯、产品消费等；

按数据类型可以划分为:

  • 动态数据: 三大维度：场景+媒体+路径

    • 场景主要包括访问设备、访问时段
    • 媒体指某一时段下用户具体访问的媒体，如资讯类、视频类、游戏类、社交类等
    • 路径指用户进入和离开某媒体的路径，可以简单理解为用户的站内与站外行为

  • 静态数据: 五大维度：人口属性+商业属性+消费特征+生活形态+CRM

    获取方式中数据挖掘是最为常见，如果数据有限，则需要定性与定量结合补充，定性方法如小组座谈会、用户深访、日志法、Laddering 阶梯法、透射法等

6. 构建标签

用户画像标签区分不同层次, 在构建时我们根据业务, 标签系统的目的, SOP(standard operation procedure)按不同层次从底层标签逐步向高层标签构建,

  • 首先, Data Clean: 缺失值判别, 聚类填充, 格式转化, 异常值处理等
  • 其次, Statistical Label: 基于清洗过的数据, 使用统计方法得出性别, 频率等标签;
  • 再次, Modeling Label: 基于ML方法计算忠诚度, 用户价值, 兴趣等标签;
  • 最后, Weights: 权重标签 = 时间衰减 X 地点权重 X 行为权重, 详见下.

7. 别忘了权重

用户画像通常以标签落地，标签也是特征工程的一种。但是用户画像的标签上有一些自己的独特之处：

  • 用户画像的标签是参数化标签

    因为业务的随时变化, 在90%的业务场景下，用户画像的标签会被改为参数化, 业务人员使用时自定义参数, 以增加标签的灵活性和适应性。

    例:「偏好电影」：过去N个月看过M场电影的人群。

  • 用户画像的标签是有权重的

    用户画像的标签务必、务必、务必记得每个标签都要有权重。如果没有权重，那么用户画像一无是处，和你们丢进碎纸机的废纸别无二致。

8. 评价指标

用户画像的评估指标主要是指准确率、覆盖率、时效性等指标。

  • 准确率指的是被打上正确标签的用户比例;

  • 覆盖率指的是被打上标签的用户占全量用户的比例，我们希望标签的覆盖率尽可能的高;

    • 覆盖率和准确率是一对矛盾的指标，需要对二者进行权衡。

  • 时效性要求不一, 对于不同的标签，需要建立合理的更新机制，以保证标签时间上的有效性。
