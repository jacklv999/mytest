过拟合和欠拟合

过拟合

  • 线性: 在训练集上表现很好， 验证集上表现很差。

  • 原因

    • 训练集的数量级和模型的复杂度不匹配。训练集的数量级要小于模型的复杂度；
    • 训练集和测试集特征分布不一致；
    • 样本里的噪音数据干扰过大，大到模型过分记住了噪音特征，反而忽略了真实的输入输出间的关系；
    • 权值学习迭代次数过多多(Overtraining)，拟合了噪声;

  • 解决方案

    • simpler model structure: 调小模型复杂度以适合训练集数量级;
    • data augmentation: 训练集越多，过拟合的概率越小;
    • 正则化: 保持模型简单，约束模型的特性;
    • dropout;
    • early stopping;
    • 交叉验证;
欠拟合

  • 现象: 在训练集上表现很差， 在验证集上面的表现也很差。

  • 原因

    • 训练的模型太简单，最通用的特征模型都没有学习到;
    • 模型复杂度过低、特征量过少等;

  • 解决方案

    • 做特征工程，添加多项式或高次特征项。
    • 使用集成方法: Bagging, Boosting;
    • 使用非线性模型;

集成算法

boosting

  • AdaBoost(Adaptive Boosting): 对数损失函数

    AdaBoost是模型为加法模型、损失函数为指数函数、学习算法为前向分步算法的二类分类学习方法，是boosting的特例.

    • 含义

      • 加法模型:

        • 先训练 Fm−1, Fm=Fm−1+fm;
        • 在旧模型上添加新的弱模型以构成强模型;

      • 前向分布算法:

    • 步骤

      • 等权重初始化训练数据的权值分布：1/N；

      • 迭代训练弱分类器: (loss func为指数损失)

        • 若某样本分类准确，降低其在新训练集的权值；
        • 若某样本分类错误，增加其权值。
        • 然后用新的样本集训练下一分类器；

      • 将各弱分类器加权投票组合成强分类器；

    • 优点:

      • 可以将不同的分类算法作为弱分类器；
      • AdaBoost具有很高的精度；
      • 相对bagging, AdaBoost充分考虑每个分类器的权重;

    • 缺点:

      • 数据不平衡导致分类精度下降；
      • 训练比较耗时，每次重新选择当前分类器最好切分点；

  • Gradient boosting: 绝对损失函数(|y(i)−f(x(i))|);

    • GBM

      • 训练步骤

        • 初始化弱学习器 f0, 此时 GBM 为 F0=f0;
        • 计算残差: Δ=y−F0(x)(loss func为绝对损失);
        • 根据残差训练新学习器 f1, 得到F1=f0+f1;
        • 不断迭代直到收敛;

      • 优点

        • 能处理缺失数据, 且对于噪声数据不敏感；
        • 能拟合复杂的非线性关系；

      • 缺点

        • 顺序计算；
        • 设置参数较多；

    • GBDT

      • 步骤

        • 初始化弱学习器 f0, 此时 GBM 为 F0=f0;
        • 计算伪残差: res=−[∂L(yi,f(xi))∂f(xi)]f=fm−1;
        • 根据残差训练新学习器 f1, 得到F1=f0+f1;
        • 不断迭代直到收敛;

      • 损失函数

        • 二分类和多分类的损失函数都是log loss;
        • 回归问题使用平方损失;

      • 特点

        • 非线性变换比较多，表达能力强，不需要做复杂的特征工程和特征变换。
        • 串行过程，不好并行化;

    • XGBoost: ergo X (Extreme) GBoosted

      • 使用CART作为基分类器时，XGBoost显式地加入了正则项;
      • GBDT只使用代价函数的一阶导数，XGBoost同时使用一阶和二阶导数;
      • GBDT采用CART作为基分类器，XGBoost支持多种类型的基分类器，比如线性分类器。
      • GBDT在每轮迭代时使用全部的数据，XGBoost则采用了与随机森林相似的策略，支持对数据进行采样;
      • 传统的GBDT没有设计对缺失值进行处理，XGBoost能够自动学习出缺失值的处理策略。

    • LightGBM

      lightGBM在传统的GBDT算法上进行了如下优化：

      • 基于Histogram的决策树算法

        • 根据直方图选择最优切割点

      • 单边梯度采样(GOSS)：计算信息增益时强调高梯度的数据;

      • 互斥特征捆绑(EFB)：将多个互斥特征绑定为一个特征以降维;

      • 带深度限制的叶子生长策略：降低复杂度;

stacking

  • 利用训练集训练得到多个初级学习器，然后用初级学习器对测试集进行预测，并用输出值和原始标签训练次级学习器;

  • sklearn库中暂时还没有支持Stacking算法的类

  • 缺点:

    • 不适合小数据: 拆分训练集和测试集可能过小;
    • 次级学习器建议用LR等简单的, 有助于减轻过拟合;
    • 不同层需要固定K-fold, 否则可能造成过拟合;

降维算法

线性方法

  • SVD降维

    • 过程: 对输入矩阵做 A=UΣVT , 得到取前 k 个奇异值的新数据集 UkΣk;

  • PCA: 与SVD类似, 但是SVD在原始数据上分解, PCA在协方差矩阵上分解;

    • 步骤:

      • 对 X 中心化后计算协方差矩阵 C=1mXXT, 并求特征值和特征向量;
      • 按特征值对应向量从大到小排列, 取前 k 行构成 P, 新数据为 Y=PX;

    • 注: Sk-learn的PCA即是用SVD求解的

    • 优点：1）最小误差。2）提取了主要信息

    • 缺点：1）计算协方差矩阵，计算量大

  • LDA: 线性判别算法

    • 原理: 将原数据(用线性回归)投影至低维空间，尽量使同类的数据聚集且不同类数据分散;

    • 步骤:

      • 计算类内散度矩阵 Sb;
      • 计算类间散度矩阵 Sw;
      • 对 Sw−1Sb 特征分解, 取前 d 个特征向量构成 W, 新数据为 Y=WTX;

    • 优点：

      1. 可以使用类别的先验知识
      2. 以标签， 类别衡量差异性的有监督降维算法， 相对于PCA的模糊性， 其目的更明确， 更能反映样本间的差异;

    • 缺点：

      1. LDA不适合对非高斯分布样本进行降维
      2. LDA最多降到K-1维
      3. LDA可能会过度拟合数据
      4. LDA在样本分类依赖方差而不是均值, 降维效果不好;
非线性方法

    • 局部改线嵌入(LLE): 假设数据中每个点可以由其近邻的几个点重构出来。降到低维，使样本仍能保持原来的重构关系，且重构系数也一样;
    • 拉普拉斯特征映射 (LE): 将数据映射到低维，且保持点之间的（相似度）距离关系。

评估模型的指标

分类模型
  • 查准率/precision: TP/(TP+FP);
  • 查全率/recall: TP/(TP+FN);
  • 准确率/accuracy: (TP+TN)/(all);
  • f-score: 当 β=1 时称为 F1 score, 当认为查准率更重要时, 减少 β, 反之减少;
F-score=(1+β2)×查准率×查全率β2×(查准率+查全率)

  • ROC曲线: 是反映敏感性和特异性连续变量的综合指标，roc曲线上每个点反映着对同一信号刺激的感受性。

    • 横坐标：1-Specificity，伪正类率(False positive rate， FPR)，预测为正但实际为负的样本占所有负例样本的比例；
    • 纵坐标：Sensitivity，真正类率(True positive rate， TPR)，预测为正且实际为正的样本占所有正例样本的比例。

  • AUC: 被定义为ROC曲线下的面积;

回归模型
  • 平均绝对误差MAE;
  • 平均平方误差MSR和RMSE;
  • 决定系数(R-square);
聚类模型

  • Adjusted Rand Index(调整过的兰德指数): 校正RI无法使随机划分的聚类结果的RI值接近0, 取值[-1,1], 越大越好.

    RI=a+bCn2,其中a表示正确聚类数,b表示错误聚类数ARI=RI−E(RI)max(RI)−E(RI)

  • 互信息: 来衡量两个数据分布的吻合程度;

  • 轮廓系数: 取值[−1,1]，同类样本越近且不同类样本越远，分数越高;

特征选择、特征创造、相关性分析这种是怎么做的？

特征选择

  • Filter: 自变量与目标变量之间的关联

    • 皮尔森相关系数（衡量变量间的线性相关性）;

    • 卡方检验: 检测随机变量与类别Y的依赖关系，删除与类别相关性小的特征;

    • 信息增益与信息增益比: 得知特征A的信息后使类Y的不确定性减少的程度(按特征取值数加权的增益比)：

      g(Y,A)=H(Y)−H(Y|A)

    • MIC，即最大信息系数(基于互信息);

  • wrapper: 通过目标函数来决定是否加入新变量

    • 迭代产生特征子集, 再评价(完全搜索/随机搜索等);
    • 递归特征消除(RFE): 递归删除特征构建模型, 基于交叉验证选择最优特征集;

  • Embedded: 学习器自身自动选择特征

    • 正则化方法（L1，L2）;
    • Feature importance: 移除该feature后模型预测错误增加量;
    • 熵, 信息增益
特征创造
  • 对于数值特征，可以尝试加减组合（当然不要乱来，根据特征表达的含义）或者提取一些统计特征;
  • 对于类别特征，我们一般会尝试之间的交叉组合，embedding也是一种思路;
  • 对于时间特征，非常重要, 但与业务相关, 如电商RFM;
