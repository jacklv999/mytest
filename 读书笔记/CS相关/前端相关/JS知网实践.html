JS知网实践

知网的很多文章只能下载CAJ格式, 但是我非常厌恶CAJ格式, 所以需要经常重新打开CNKI的英文版, 然后重新搜索下载, 非常麻烦. 此次我做了一个自动化, 自动打开对应文章的CNKI版页面.

相关代码用书签打开, 简单代码如下:

x
javascript:(
    function(){
        var tmp = window.location.href;
        temp=tmp.replace(
            "kns/kcms/",
            "oversea/KCMS/"
        );
        new_url=temp.split(
            "&"
        )[0]+"&"+
            temp.split(
            "&"
        )[1]+"&"+
            temp.split(
            "&"
        )[2];
        window.open(new_url);
    }
)();

以上代码主要干了这么几件事:

  • 获取当前页面链接
  • 拼接对应英文版链接
  • 跳转英文版链接

接下来依次说明

1. 获取当前页面链接

用 JavaScript 获取当前页面链接可以直接使用函数 window.location.href 获取, 示例如下:

xxxxxxxxxx
var tmp = window.location.href;

类似的函数还包括:

  • window.open: 作用见下文
  • window.onload: 在页面加载完成后执行代码
2. 拼接对应链接

拼接涉及 JavaScript 的字符串操作, 以下记录对应字符串操作

  • 分割字符串:

    • 按符号: str.split("&")

      • 分割字符串的结果是列表
      • js 列表用 list[index] 访问

    • 按长度: s.substring(0,5)

  • 拼接字符串: str_1 + str_2

3. 跳转链接

JavaScript 中跳转链接有两种方式, 看自己需要选择

  • window.location.href="new_url"

    • 当前窗口跳转新链接

  • window.open(new_url)

    • 新窗口打开链接

附录

1. 利用 jQuery 等跳转页面

使用 JQ 等跳转页面更加方便, 主要包括 $.get 和 $.ajax 两类方法. 但是也存在限制, 即不能跨域跳转(CORS).

当然, 限制也是可以解除的, 需要在 header 中添加 Access-Control-Allow-Origin:'*' , 但是在我的试验中, 均未成功.
